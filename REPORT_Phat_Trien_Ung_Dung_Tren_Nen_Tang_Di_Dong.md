**1.4 Giới thiệu nền tảng phát triển ứng dụng di động**
- Tổng quan về nền tảng: Nền tảng phát triển di động là tập hợp công cụ, framework và thư viện hỗ trợ tạo ứng dụng chạy trên thiết bị di động, đảm bảo hoạt động mượt mà, tương thích đa thiết bị và dễ bảo trì.
- Kiến trúc phát triển ứng dụng di động: Ứng dụng được phát triển theo kiến trúc MVVM (Model-View-ViewModel) kết hợp với các thành phần Jetpack để tách biệt giao diện, logic và dữ liệu; sử dụng Coroutines và Flow cho bất đồng bộ.
- Các công cụ hỗ trợ phát triển: Android Studio làm IDE chính; Kotlin là ngôn ngữ; Jetpack (ViewModel, Room, Navigation); Coil cho ảnh; Gradle/KSP cho build; database sử dụng MongoDB cho backend (hoặc lưu cục bộ tùy mô hình).
- Đặc điểm và nguyên tắc hoạt động: Ứng dụng trên thiết bị di động phải tối ưu bộ nhớ/CPU, xử lý offline/online, bảo mật dữ liệu người dùng (mã hóa, quản lý session) và tuân thủ nguyên tắc responsive UI, lifecycle-aware components.

### Chương 2: Phân tích hệ thống
- **Các tác nhân (Actors):**
  - Khách hàng (User)
  - Quản trị viên (Admin) — phần Admin có thể được mở rộng trong tương lai.
  - Hệ thống (ứng dụng)
- **Các nghiệp vụ (Use Cases) chính:**
  - Xem danh sách phòng/chi tiết phòng
  - Tìm kiếm theo địa điểm, tên, bộ lọc
  - Đăng ký tài khoản, đăng nhập
  - Thêm/xóa phòng yêu thích
  - Tạo/huỷ booking
  - Quản lý phòng (Admin — thêm, sửa, xoá)
- **Biểu đồ Use Case / Tuần tự (Sequence diagrams):**
  - Người dùng tìm kiếm phòng → hệ thống truy vấn DB → trả kết quả
  - Người dùng đăng ký/đăng nhập → hệ thống validate → lưu user trong DB
  - Người dùng chọn phòng → chọn slot/ngày → đặt phòng và lưu booking
- **Thiết kế cơ sở dữ liệu:**
  - Entities chính:
    - `Room`: thông tin phòng (tên, mô tả, địa điểm, giá, ảnh, isAvailable, ...)
    - `Slot`: mô tả các slot/giường/loại phòng trong `Room`
    - `User`: tài khoản người dùng (email, số điện thoại, password — lưu cần bảo mật)
    - `Booking`: lưu thông tin đặt phòng (userId, slotId, fromDate, toDate, trạng thái)
    - `Favorite`: lưu phòng yêu thích (userId, roomId)
  - Sử dụng Room ORM (SQLite) cho việc quản lý dữ liệu cục bộ.
  - Thiết kế ERD: mối quan hệ 1-N giữa Room và Slot; 1-N giữa User và Booking; 1-N giữa User và Favorite.

### Chương 3: Triển khai hệ thống
- Môi trường: Android Studio, SDK, Gradle, KSP; chạy debug trên emulator hoặc thiết bị thật.
- Kiến trúc: MVVM (View - ViewModel - Repository - Room DAO), Coroutines + Flow, Coil.
- Cơ sở dữ liệu: Room (`HomestayDatabase`) với entities Room/Slot/User/Booking/Favorite; `DatabaseSeeder` seed dữ liệu trong `HomestayApplication.onCreate()`.
- Thành phần chính: `MainActivity` (danh sách & tìm kiếm), `RoomDetailActivity` (chi tiết & booking), `LoginActivity`/`RegisterActivity` (auth), `Profile`/`Booking`/`Favorite`.
- Kiểm tra & debug: Android Studio Database Inspector, Logcat, hoặc pull DB bằng `adb` để mở bằng DB Browser for SQLite.
